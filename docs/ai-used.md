# AI 활용 내역

AI를 단순 코드 생성 도구가 아니라 도메인 이해, 설계 검토, UI 기획, 구현 오류 해결, 문서화 보조 도구로 활용.
AI가 제안한 내용을 과제 요구사항과 실제 데이터 흐름에 맞는지 검토한 뒤 필요한 부분만 선별 적용.

---

## 1. 도메인 이해 및 요구사항 정리

**목적**

PCF(Product Carbon Footprint), GHG Scope, 배출계수, 활동 데이터처럼 익숙하지 않은 탄소 회계 개념을 빠르게 정리하고, 과제 요구사항을 실제 기능 단위로 해석하기 위해 사용

**활용전략**

먼저 AI에게 제조/물류 SaaS 관점에서 PCF 계산에 필요한 데이터와 GHG Scope 1, 2, 3의 차이를 설명하게 했습니다. 이후 실무자와 경영자가 각각 어떤 정보를 필요로 하는지 구분하고, 과제 체크리스트를 화면과 기능 요구사항으로 재정리했습니다.

```text
탄소 관리 플랫폼을 만들건데 PCF, GHG Scope, 배출계수 등 
대시보드를 구현하기 전에 필수적으로 알아야하는 핵심 개념들을 상세히 설명해줘
```

**결정근거**

탄소 회계 용어는 익숙하지 않은 도메인이라, 처음부터 구현에 들어가면 잘못된 전제를 코드와 화면에 반영할 위험이 크다고 판단. 그래서 AI를 먼저 개념 정리와 요구사항 해석에 사용

---

### 시스템 설계 및 기술스택 선정

**작업내용**
- 기술 스택 후보군 교차 검증: 기술 호환성 및 설정 리스크 검토 지시.
- DB 스키마 베이스라인 생성: 탄소 회계 도메인에 필요한 기본 테이블 구조에 대한 초안 가이드 생성 및 성능 검증.
- 기술 스택 전략 수립: 기능 구현 범위 지정 및 과제 요구사항에 맞는 초기 파일구조 자동 생성

**활용전략**
- 도메인 모델링 베이스라인 요청
- 스캐폴딩 및 범위 구체화

```text
README.md 파일을 기반으로 프로젝트 개발 환경을 구축 해주고 라이브러리 간 버전 충돌 가능성이나 컨테이너 빌드 최적화 측면에서 주의해야 할 체크리스트를 알려줘

PCF 산출을 위해 제품, 활동 데이터, 배출계수 간의 관계를 정의하는 스키마 초안을 작성

주어진 과제 요구사항을 바탕으로 적절한 스케일의 폴더 구조 제안 및 보일러플레이트 작성
```

**결정근거**
- 인프라 리스크 사전 차단: 
- 데이터 설계의 확장성 확보:
  - 직접 선정한 기술 스택의 잠재적 결함(Connection Leak, 컨테이너 오버헤드 등)을 AI로 교차 검증하여 환경 구축 시간을 단축하고 운영 안정성 확보.
- 데이터 설계의 확장성 확보:
  - AI가 제안한 기초 뼈대를 바탕으로 버전 관리와 원본 데이터 보존 등 실무적 세부 사항을 직접 덧붙여 도메인에 최적화된 DB 구조 확립.
- 개발 생산성 극대화:
  - 프로젝트 초기 설정 및 폴더 구조 생성 등 반복적인 보일러플레이트 작업을 자동화하고, 구현 범위에 대한 객관적 우선순위를 검토하여 제한된 시간 내 고품질 산출물 도출.

### 대시보드 UI 기획

**작업내용**
- 경영자와 실무자 관점에서 필요한 정보와 화면 우선순위 정리.
- PCF 결과를 단순 수치가 아니라 Scope, 배출원, 월별 추이로 해석할 수 있는 대시보드 구조 검토.
- Excel 업로드 후 검증·수정·저장 흐름이 실무자 화면에서 자연스럽게 이어지도록 UI 흐름 정리.

**활용전략**
- Claude Design과 ChatGPT를 함께 사용하여 화면 구성 아이디어를 빠르게 탐색.
- 경영자, 실무자 역할을 나눠 Prompt를 작성하고 각 사용자 관점에서 필요한 정보를 도출.
- 여러 Prompt에서 반복적으로 등장한 지표를 우선순위 후보로 두고, 실제 구현 범위와 비교하여 최종 구성 결정.

```text
탄소 발자국 관리 SaaS의 경영자 대시보드를 만든다면
30초 안에 현황을 파악할 수 있도록 어떤 KPI와 차트가 필요할까?
```

```text
실무자가 엑셀로 탄소 활동 데이터를 업로드하는 화면을 설계하려고 해.
업로드, 컬럼 매핑, 유효성 검사, 오류 수정, 최종 저장 흐름을 UX 관점에서 정리해줘.
```

**결정근거**
- 탄소 회계 도메인에 대한 초기 인사이트가 부족했기 때문에, 먼저 AI를 통해 사용자별 니즈와 대시보드 후보 지표를 넓게 수집.
- Prompt에 “경영자”, “실무자”, “30초 안에 현황 파악”, “Excel 업로드 흐름”처럼 역할과 사용 상황을 명시하여 일반적인 차트 추천이 아니라 과제 맥락에 맞는 답변을 유도.

### 개발 - API 및 데이터 연동

**작업내용**
- 새로나온 Claude Design을 활용해 초기 화면에 디자인 시스템을 빠르게 적용.
- API Query 및 데이터 집계 로직 작성 보조.
- Swagger 문서 주석과 API 응답 구조 정리.
- Seed 데이터와 Excel 업로드 검증용 테스트 데이터 후보 생성.

**활용전략**
- 정해진 Input/Output 구조를 먼저 정의한 뒤, 필요한 Prisma query와 데이터 매핑 로직 생성에 활용.
- 이미 구현된 API 응답 구조를 기준으로 Swagger 주석 초안을 생성하고, 실제 코드와 맞지 않는 부분은 직접 수정.
- 과제 데이터와 유사한 원소재·전기·운송 활동 데이터를 기준으로 seed 데이터 후보를 생성.
- Excel import 과정에서 발생할 수 있는 정상/경고/오류 케이스를 AI에게 먼저 도출하게 한 뒤, 핵심 플로우 위주로 구현 범위 조정.
- Claude Design을 통해 카드, 테이블, 차트, 입력 영역의 시각적 톤을 빠르게 잡음.

```text
아래 API 응답 구조를 기준으로 next-swagger-doc에서 사용할 수 있는
OpenAPI 주석 초안을 작성해줘.
```

```text
PCF 대시보드 seed 데이터를 만들려고 해.
제품, 생산량, 원소재, 전기, 운송 활동 데이터와 Scope 1, 2, 3 배출계수가
모두 포함되도록 샘플 데이터를 구성해줘.
```

```text
Excel 업로드 기반 PCF 데이터 입력 화면에서 검증해야 할 테스트 데이터를 생성해줘
정상 케이스, 필수값 누락, 날짜 형식 오류, 수량 오류, 배출계수 미매칭 등
```

**결정근거**
- API와 데이터 연동은 입력과 출력 구조가 비교적 명확하기 때문에, AI가 반복적인 query 초안과 문서 주석을 빠르게 만드는 데 효과적이라고 판단.
- 테스트 데이터는 대시보드 차트와 PCF 계산 결과가 실제로 보이는지 확인하기 위한 목적이므로, Scope별 데이터와 활동 유형이 고르게 포함되도록 Prompt에 조건을 명시.
- 테스트 케이스는 사용자가 실수하기 쉬운 Excel 입력 오류를 빠르게 빠짐없이 나열하기 위해 AI를 활용. 이후 실제 구현 범위에 맞춰 클라이언트 검증과 서버 검증으로 나누어 반영.
- 디자인 시스템은 처음부터 세부 스타일을 직접 잡으면 시간이 많이 걸리기 때문에 Claude Design을 활용해 빠르게 방향을 확인 및 AI Chain 활용

### 검증 및 문서화

**작업내용**
- 과제 체크리스트 기준으로 README 누락 항목 점검.
- Docker Compose, Yarn 실행 방법, Prisma 환경변수 설정 문서화.

**활용전략**
- 과제 요구사항과 README 내용을 비교하여 빠진 항목을 AI에게 점검하게 함.
- Docker/Prisma 오류 해결 과정에서 발생한 시행착오를 README 실행 방법에 반영.

README 점검 Prompt:
```text
아래 과제 체크리스트를 기준으로 현재 README에서 부족한 항목을 점검해줘.
실행 방법, 아키텍처, ERD, 스크린샷, AI 활용 내역이 평가자가 보기 쉽게 들어갔는지 봐줘.
```

**결정근거**
- Prompt에는 과제 체크리스트 항목을 직접 넣어, 일반적인 README 조언이 아니라 제출 조건 기준의 검토 결과를 얻고자 함.
- AI 활용 내역은 README에 모두 넣으면 본문이 길어지기 때문에 별도 문서로 분리. 대신 README에서 연결할 수 있도록 구성해 평가 항목을 놓치지 않게 정리.
